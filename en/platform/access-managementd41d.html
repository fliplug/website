User authentication

Balance security with user experience

Design and enforce strict—yet user-friendly—authorization protocols and flows for your applications and APIs.

No matter what kind of authentication and onboarding flows you want to design, fliplug offers a solution. 

fliplug currently supports the following auth protocols:

  • OAuth2
  • OpenID Connect
  • UMA 2.0
  • SCIM 2.0
  • FAPI 1.0
  • CIBA 1.0
  • SAML 2.0

Centrally manage federated third party identity providers and even use fliplug Access Management to create your own identity provider. 

With over 15 supported identity providers, you’ll have no problem building a federated identity management strategy around the fliplug Access Management solution.

Explore more Book a demo

fliplug Access Management supports W3C Web Authentication (WebAuthn), allowing users to authenticate their accounts without a password.

WebAuthn is supported in the Chrome, Firefox, and Edge browsers to different degrees, but support for credential creation and assertion using a U2F Token, such as those provided by Yubico and Feitian, is supported by all of them. 

This enables you to design and enforce strict security measures, while still giving your users the smoothest onboarding experiences possible.

Explore more Book a demo

You can use flows to extend the standard fliplug Access Management (AM) behavior by executing policies during the OnRequest step of selected stages. Flows can be configured at the security domain level or application level.

Gravitee AM supports flow design for the following stages:

  • All stages: authentication flows are executed for every user request
  • Login: execute specific flows before and/or after user login
  • Consent: execute authentication policies before or after displaying user consent forms
  • Registration: execute flows before or after users are given registration forms

Multi-factor authentication

Implement top-notch MFA experiences

Strengthen your security posture with user-friendly MFA options, all supported out of the box by fliplug Identity and Access Management.
See it for yourself

Multi-factor authentication (MFA) provides a way to add additional steps during the login flow to enforce access control. This ensures that only valid users can access their accounts even if their credentials have been compromised.

fliplug supports several different authentication factors, such as email, one-time-password, biometric factors, recovery codes, and more.

Explore more Book a demo

One of the major challenges in implementing MFA is poor user and onboarding experience. Constantly forcing users to authenticate themselves over and over again can seriously disrupt the user experience, forcing even what should be trusted users to use multiple tools, devices, etc. to log in to commonly used applications.

Conditional MFA allows you to define fine-grained conditions for MFA flows to be executed. Now, you can enforce extra authentication factors if and when necessary, allowing for a perfect balance of security and customer experience.

Explore more Book a demo

One of our various forms of conditional authentication, step-up authentication, deserves standalone promotion. 

With step-up auth, you can spread MFA across multiple steps in your product, only forcing extra factors of authentication when users try to access sensitive APIs or functionality.

For example, you could force a single authentication factor at the login phase but then add an extra biometric authentication factor when users try to do something sensitive in the application, such as wiring funds or viewing account balances.

This is the ultimate MFA experience, ensuring easy, proper onboarding for users while still enforcing strict security where it absolutely needs to be enforced.

Explore more Book a demo

Risk-based MFA leverages conditional logic, monitoring data, and intelligent processing of that data to alert fliplug Access Management when extra authentication factors need to be enforced.

You can use fliplugto configure risk assessments based on device location, IP reputation score, and geolocation velocity (i.e., how quickly devices seem to change physical location when trying to access an application or a service). Then, you can configure Gravitee AM to enforce specific authentication factors when a user is deemed too high-risk based on intelligent risk assessments.

API Security

IAM + the fliplug API Gateway = advanced API security

Cover your every API Security need by combining the security functionality of both fliplug's API Gateway and the fliplug IAM solution.
See it for yourself

Working with an API Gateway to enforce API authorization, such as OAuth2 API auth? 

fliplug Access Management can be used as an OAuth2 authorization resource by your API Gateway. This essentially means that, when your API Gateway enforces OAuth2 authorization at the moment of an API request, that Gateway will “call” fliplug Access Management for credential checking.

This is even more powerful when combining fliplug Access Management with the fliplug API Gateway . fliplug Access Management is a preconfigured authorization resource and can be configured as your auth provider in a matter of a few clicks.

Explore more Book a demo

When publishing your APIs to your fliplug API Developer Portal, you can Access Management to control access to both:

  • the fliplug Developer Portal
  • Specific API subscriptions

This is done by managing authentication and access for fliplug API Management users via fliplug Access Management and by  backing fliplug API Management OAuth2 subscription plans as the OAuth2 authorization resource.

Explore more Book a demo

When you pair fliplug Access Management With fliplug API Management, you can apply all the benefits of AM functionality, such as step-up authentication, at the API call level.

For example, you can use fliplugAPI Management to define authorization flows and policies that the API Gateway will execute. The fliplug API Gateway will then leverage fliplug Access Management to enforce things like step-up authentication when users make certain API calls.
