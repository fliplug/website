About Gravitee API Management

The Gravitee event-native API Management platform enables teams to manage, secure, and expose synchronous and asynchronous APIs. Our event-native foundations give the Gravitee Gateway (yes, a single Gateway) the flexibility to:

  • Support of all modern AsyncAPI protocols (i.e. Websocket, Webhook, SSE)
  • Provide native integration to event brokers (e.g. Kafka, Kinesis, Solace, RabbitMQ, MQTT, JMS)
  • Fully support the AsyncAPI specification across the entire lifecycle of the API (i.e. API Designer support AsyncAPI specs and so does the portal)
  • Apply traditional APIM capabilities (policy execution, traffic shaping, quotas, rate limiting, authentication, Analytics, etc.) to asynchronous and synchronous APIs

In addition to synchronous and asynchronous flexibility, Gravitee offers a wide breadth of functionality that spans the entire API lifecycle—offering a graphical API Designer, a CLI-and-UI-enabled Gateway and Management console, API Security and Access Management, a fully-featured API Developer Portal, API Monitoring, and more.

If you want to learn more about how Gravitee stacks up to other, non-event-native solutions, check out the in-depth comparisons of Gravitee vs Tyk below!

The TLDR: Gravitee API Management vs Tyk API Management

About Gravitee and API Management

Gravitee provides an event-native API Management platform and Gateway. Gravitee OSS products are more fully-featured, and they come with a UI. Beyond OSS, Gravitee’s enterprise solution offers much more in-depth support for synchronous and asynchronous API use cases. For example, Gravitee offers support for Kakfa, Webcoskets, Webhooks, Server-sent events, etc. in addition to support for REST and SOAP use cases. Gravitee can even mediate between REST and Kakfa, REST and Webhooks, etc. while applying policies to requests, responses, and messages.

About Tyk and API Management

Tyk has a mature open source API gateway solution. The open source edition provides a CLI, but no UI. Enterprise users do have a UI available, but this costs extra money. Tyk is also developed in Go so it may suit Go environments and Dev teams more than Kong would.

Like Gravitee, Tyk has a paid Developer Portal option available. Where Tyk may have Gravitee, and other API Management vendors, beat is their rich support for GraphQL. Tyk’s Universal Data Graph is a very technical solution that allows teams to create a connection between GraphQL and and other existing infrastructure or service that you currently have within your organization. So, for example, you could connect a GraphQL endpoint in Tyk that you connect to a RESTful API that you are using as a data source. However, Tyk only supports REST and SOAP as data sources for the time being. Gravitee also has rich support for GraphQL on our roadmap.

About Gravitee and API Access Management & Security

Gravitee offers a fully-featured Identity and Access Management solution that you can use for both application and API Access control and Security. By applying IAM to both applications and APIs, you are able to achieve true last mile security, where you are able to control and secure access to API-specific communication methods. Gravitee comes with security-focused policies baked into the Gravitee API Gateway and API Management solution.

About Tyk and API Access Management & Security

While Tyk has a mature API gateway, they do not have a dedicated access management solution. Most typically, organizations would deploy Tyk along with a dedicated access management solution such as NoName, Auth0, Okta etc. Where Tyk might have an edge is its ability to potentially use its Universal Data Graph and API Management capabilities to partially secure GraphQL-connected services. Other than this, Tyk simply cannot match Gravitee’s Security offerings given that Gravitee is the only API solutions vendor that also offers dedicated Identity and Access Management.

About Gravitee and API Design

Gravitee provides an API Design solution that makes API Design and Development effortless, governable, and secure with a no-code, drag n’ drop mind map interface. This ease of use makes the Gravitee API Designer more suitable for less technical users like Business Analysts, Product Owners, API Product Managers, etc. Because of this, the Gravitee API Designer is very well suited for an API Design-first approach.

Tyk and Gravitee both offer API Design solutions, but Gravitee’s API Designer gets the edge for less technical teams and teams who are interested in implementing an API Design-first approach to API Design and application development.

About Gravitee and API Developer Portal, API Productization, and API Monetization

Like Tyk, Gravitee offers an API Developer Portal that can help with productizing your APIs. However, Gravitee differs from Tyk on a few fronts.

On the positive side, we offer a Developer Portal in our OSS edition. Also, our enterprise-grade Developer Portal supports asynchronous/streaming APIs. This makes it easy to expose and potentially drive revenue with both synchronous and asynchronous APIs. We also support applications, developers, and consumers in the Developer Portal.

On the negative side, some teams will prefer Tyk’s advanced support for GraphQL in a Developer Portal, and, to be fair, Tyk has everybody beat (out of the vendors we have compared) here.

Our final verdict is that Gravitee is going to be a more suitable API Developer Portal and API Productization solution for most teams, with the exception being teams that are fully committed to GraphQL. And, even that recommendation is limited, as Gravitee currently has advanced support for GraphQL on the roadmap.

Like Gravitee, Tyk offers an API Developer Portal that can help with productizing your APIs.

We give Gravitee the edge on the API Developer Portal and API Productization play for most teams, as Gravitee offers a Developer Portal in its OSS option and supports applications, Developers, and Consumers in Dev Portal.

However, if you are committed to GraphQL, Tyk might be the better solution (for now), as their Developer Portal offers support for GraphQL, whereas Gravitee’s does not yet.

Our maybe-biased take on Gravitee vs Tyk

Tyk is an interesting API Management solution. Like Gravitee, Tyk is very clearly focused on building API-related solutions for more modern use cases, whereas vendors like Kong and Apigee are clearly more focused on niche use cases (such as Service Mesh for Kong) for more present-moment synchronous API use cases. 

However, the modern use cases that Gravitee and Tyk seem focused on are different. Tyk seems to have fully-committed to GraphQL and seems set on offering more advanced GraphQL support than other API Management vendors. 

On the other hand, Gravitee has chosen to take the route of flexibility and breadth. By flexibility, we mean the ability to support a large number of modern use cases by building an event-native API Gateway and management console that supports a wide variety of asynchronous, streaming, and event-driven API use cases, instead of just focusing on a single technology like Tyk’s focus on GraphQL. In addition to flexibility, Gravitee’s breadth refers to Gravitee’s offering API Management, IAM, API Security, API Design, API Developer Portal, API Monetization, and more API-related solutions.

We feel that Tyk has gone the route of being an Integration platform utilizing GraphQL as Middleware to enable transformations between different protocols and offering the result as a query. For enterprises that are dead set on using GraphQL, Tyk might be the better option (for now). However, Gravitee has chosen a broader, more flexible approach with solutions that help you manage, secure, and govern synchronous and asynchronous API ecosystems built on a variety of different protocols and services.
