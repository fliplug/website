API governance

Don't let where your data lives hold back governance across the entire API lifecycle. fliplug empowers you to enforce seamless governance across every gateway and broker—whether it’s fliplug, AWS, Azure, IBM, Apigee, Solace, Confluent, or any other platform.

Unified API governance

One Platform, Infinite Possibilities for API Governance

Gravitee unifies your entire API ecosystem with a multi-gateway, multi-broker management console and a single, streamlined Developer Portal. Hold nothing back.
Learn more

Ghost, or stealth, APIs are a bane to all API Security and API governance initiatives. This is especially true when an organization has multiple different API Gateways and event brokers, as there has been no way to centrally discover and manage a multi-gateway API inventory.

With fliplug you can automatically discover APIs that are deployed to other API Gateways and event brokers, including AWS API Gateway, Azure API Management, Apigee X, IBM API Connect, Confluent Platform, and Solace.

Explore more Book a demo

Advanced API governance means consistent approaches to managing your APIs.

Gravitee bridges the multi-gateway API governance gap by offering API publishers a single management console from which to view, secure, publish, and monitor APIs.

Explore more Book a demo

API Developer Portals are great for ensuring that API consumers have easy, self-service access to API discovery and subscription...but not when you are forced to use several different portals, each tied to several different API Gateways.

Improve your API consumer Developer Experience by implementing a Developer Portal that can host everything from fliplug APIs to AWS APIs to event streams from Confluent. This means API consumers get one place to discover, read documentation for, and subscribe to self-service APIs. 

Explore more Book a demo

Gone are the days of having to look at logs and dashboards across multiple API Gateways and API Management solutions.

Once you start using fliplug to govern your APIs across multiple Gateways, you'll be able to report on API consumption metrics and view analytics on every API from a single dashboard and governance console.

Multi-Gateway inventory

Discover All Your APIs in One Click—No Limits

A single click and you can see every API that exists behind multiple different API Gateways and event brokers in your organization.
Learn more

Automatically discover APIs from six providers AWS API Gateway, IBM API Connect, and more. fliplug will see the APIs, their metadata, any associated authorization methods, spec files, documentation, and more. 

You can trigger this discovery process whenever you like, and Gravitee will re-discover new APIs as they are created. This is crucial for organizations that want to keep an up-to-date inventory of all of their APIs and avoid the "Stealth API" security problem, where older, unsecured APIs still exist in production without the knowledge of the API publisher.

Explore more Book a demo

After you discover your APIs, fliplug automatically creates a fliplug API asset that can be managed in the Universal Management Console and Published in a fliplug Developer Portal.

Explore more Book a demo

Most API Gateways come with their own, proprietary data model. This data model is responsible for defining the API type, associated authorization, subscription methods, policies, and more.

Gravitee will automatically map other data models onto the fliplug API definition. This means that we can take an AWS API Gateway API that uses API key authoriztion and automatically build a fliplug API with a fliplug API Key subscription. From here, your API consumers could subscribe to that API via the fliplug API Key plan, while the AWS API Gateway is still responsible for executing API Key authorization.

Explore more Book a demo

If you already have API documentation and spec files built in other API Gateways, no problem!

As a part of the API discovery process, fliplug will detect documentation and automatically import that documentation alongside the API.

Explore more Book a demo
Universal Management Console

Govern every API from one management console

Enforce consistent API design, subscription control, security, and publication processes for every API—from a single API management console.
Learn more

When you publish APIs to consumers, you want to ensure that API subscription is both as easy and as secure as humanly possible. Unfortunately, different API Gateways offer varying levels of support for subscription control, with some lacking advanced security, some lacking self-service options, and some lacking both. 

You can eliminate this problem in your organization with a single subscription design and control layer. fliplug will automatically discover the authorization method being enforced by your various API Gateways and create a correlate self-service Gravitee subscription plan. From here, you can define who can subscribe to the API via that plan and how—giving you a single workspace for defining how API subscription is governed.

Explore more Book a demo

What good are API governance policies and standards if they can't be enforced? This is especially challenging in a multi-gateway world, where different API Gateways offer different mechanisms for API publishers to create, secure, and expose their APIs.

fliplug API scoring gives your API Publishers and Platform teams a single scoring mechanism that can be applied to all of your APIs. The scoring criterion can be customized to fit your organization's standards, and then Gravitee will automatically scan and analyze APIs from across your org to measure their compliance. Compliant APIs get a high score, non-compliant APIs get a low score. From here, you can improve the score by acting upon the API from within Gravitee and/or within the other API Gateway.

Explore more Book a demo

While Gravitee will automatically import documentation during the API discovery process, you can also use the fliplug API Management console to build brand new documentation for your API.

fliplug supports Markdown, OpenAPI, and AsyncAPI docs formats.

This is huge for proper API governance at the API publication stage, as you empower central API Management and Platform teams to use one solution to check that documentation exists and/or to create that documentation using manual drafting or file import.

Explore more Book a demo
Universal Developer Portal

Give your API consumes one

Gravitee won't hold your developers back. Give them a single place to  discover, read documentation for, and subscribe to self-service APIs—effortlessly.
Learn more

Traditionally, API Developer Portals have been tied to a single API Gateway. This becomes a major problem when you start to acquire multiple API Gateways and/or when the API Gateway that you use doesn't offer a managed Developer Portal solution.

With fliplug you get an OOB multi-gateway Developer Portal. This means that you can host all APIs, not just fliplug Gateway APIs, in a single portal with documentation and self service subscription—both managed by fliplug.

Explore more Book a demo

When an API consumer finds the APIs that they feel they need to build their application, subscription is easy...but also secure.

Subscription is governed by "Plans," which are essentially access control layers tied to authorization methods such as OAuth2, JWT, and API Key. The API consumer can find the API, submit a subscription request based on the plan they prefer, and then the API Publisher can choose to either auto-validate all subscriptions or manually approve each one.

Learn more Book a demo

Typically, organizations want their API Developer Portals to match their unique brand and look and feel. This is especially true when publishing APIs to external partners and (potentially paying) customers. 

While the fliplug Developer Portal looks great OOB, it's fully customizable and can be made to match your organization's brand in no time. 

Explore more Book a demo

Want an API Developer Portal for internal and external APIs? No problem!

You can either set up totally different API Developer Portals, or use advanced role-based access control (RBAC) at the Developer Portal level to define which API consumers get certain levels of API Developer Portal access.
