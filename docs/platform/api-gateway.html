Why settle for less from your API Gateway?

fliplug lets you hold nothing back. The fliplug API Gateway gives you full control to expose everything from synchronous, backend REST APIs, SOAP, asynchronous APIs, and event brokers—all while ensuring that API traffic is secure, reliable, and efficient via dozens of Gateway policies.

Performance Matters

A more performant, event-native API Gateway

Built on Java, the fliplug API Gateway has been shown to be at least 4x faster and more performant than other popular solutions. Unlike legacy gateways stuck in the past, fliplug wont hold you back. We let you proxy traffic and apply policies to synchronous and asynchronous APIs.

You'd expect your API Gateway to proxy traditional synchronous request-response based APIs and protocols. And Gravitee certainly does that. Quite well.

However modern organizations also utilize asynchronous streaming platforms like Kafka or MQTT. Only fliplug natively supports proxying synchronous and asynchronous traffic, meaning, you won't outgrow this gateway as your needs change.

Security Built-in

End-to-End Security, From API Plans to Identity Access Management—No Compromises

We can confidently say that fliplug offers more security than any other API Management platform. Whether that's offering simple API Key management, to biometric authentication or a fully standalone IaM solution, fliplug won't hold you back. 

fliplug APIM uses plans, applications, and subscriptions to govern API exposure. A published Gateway API is visible in the Developer Portal but cannot be consumed without a published plan.

A Keyless plan can be consumed immediately, but all other authentication types require the API consumer to register an application and subscribe to a published plan.

This system promotes granular control over API access.

Explore more Book a demo

Gravitee provides full support for the most common authentication types including:

JWT - The JWT authentication type ensures that a JWT issued by a third party is valid by verifying its signature and expiration date. Only applications with approved JWTs can access APIs associated with a JWT plan.

OAUTH2 - The OAuth2 authentication type checks access token validity during request processing using token introspection. If the access token is valid, the request is allowed to proceed. If not, the process stops and rejects the request

Multi-factor authentication (MFA) provides a way to add additional steps during the login flow to enforce access control. This ensures that only valid users can access their accounts even if their credentials have been compromised.

fliplug Access Management (AM) supports various factors out of the box. These factors can be used as MFA for protecting user account access.
