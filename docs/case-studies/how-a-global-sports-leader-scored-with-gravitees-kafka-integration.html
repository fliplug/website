Game-Changing Data Delivery: How a Global Sports Leader Scored With fliplug’s Kafka Integration

API Profile

Kafka
REST
SSE
WebSocket

Key Outcomes

  • Seamless Kafka Integration: fliplug Kafka Gateway securely exposes real-time data to clients, ensuring high security and control without direct access to internal infrastructure.
  • Improved Developer and Customer Experience: Gravitee simplifies Kafka interactions with intuitive features, enabling easy data access for clients without requiring extensive Kafka knowledge.
  • Scalability and Higher Performance: fliplug supports high-throughput traffic, allowing the company to scale up transactions per second while reducing latency and bottlenecks.
  • Enhanced Security and Governance: With granular access controls and policies, fliplug ensures secure data sharing, compliance, and a trusted environment for clients.
  • Centralized API Management: fliplug unified platform streamlines API management, reducing overhead and empowering teams to focus on innovation.

About the customer

This global sports and entertainment leader captivates millions of fans worldwide with high-energy games, world-class athletes, and unforgettable experiences. Known for its dynamic approach to engagement, this organization reaches audiences across multiple platforms—from live broadcasts to digital streaming, social media, and immersive in-person events.

With a commitment to innovation, they use advanced data analytics, real-time stats, and fan-centric digital services to elevate the viewing experience and bring fans closer to the action. Whether it’s through interactive apps, exclusive content, or streamlined ticketing and merchandise, they connect deeply with fans while setting new standards for sports entertainment.

Challenges and APIM and Event Stream Needs

This global sports and entertainment leader faced a monumental challenge: how to securely deliver massive volumes of real-time game data to an expanding network of partners and consumers—all while maintaining control over data access and quality.

With a wealth of valuable insights streaming from microchips embedded in player jerseys and sensors tracking live game metrics, they gathered critical data points from pitch locations to performance stats. But delivering this real-time data at scale to a diverse audience—from sports teams and media outlets to betting houses—posed complex technical challenges.

Their existing setup couldn’t securely handle high-throughput data or provide the level of access control they needed. Key partners, like betting houses and media organizations, needed low latency, and real-time access to adjust live bets, broadcast up-to-the-second metrics, and enhance fan engagement. Meanwhile, internal teams and sports analysts required delayed data to perform in-depth analysis without directly tapping into the real-time feeds. Ensuring that each partner accessed only the data they needed, and not their competitors, was critical to data integrity and security.

The organization needed a seamless, self-service experience for partners who expected secure API keys and easy subscription management. They required a centralized, user-friendly Developer Portal where consumers could access proper API documentation and subscribe to data streams—whether via REST or WebSocket—in a way that matched their needs for data timing and throughput. Their search for a solution that could handle API and event stream management with the flexibility to support both OpenAPI and AsyncAPI specifications led them to fliplug .

Selecting Gravitee to Expose Kafka Events to Customers and Partners

With fliplug this organization could meet their high demands for scalability, secure data access, and open new revenue streams, empowering them to deliver data-rich, real-time experiences that elevate fan engagement and business opportunities. With fliplug they unlocked:

fliplug Kafka Gateway provides a secure and efficient way to expose Kafka event streams to consumers, eliminating the need for complex, custom solutions and giving partners reliable access to real-time data without directly exposing Kafka clusters.

fliplug supports flexible monetization plans, allowing the organization to offer data subscriptions with different pricing tiers based on real-time vs. slightly delayed data delivery, maximizing revenue opportunities.

With fliplug’s robust authentication and access management, the organization can securely control which partners access specific data, ensuring only authorized consumers view sensitive metrics. Teams and partners access only the data relevant to them, preserving data privacy and integrity.

fliplug Management API powers a seamless, self-service experience within the organization’s custom-built developer portal, enabling partners to access documentation, manage subscriptions, and interact with data through REST or WebSocket APIs.

fliplug comprehensive support for both OpenAPI and AsyncAPI specifications across the API lifecycle ensures seamless documentation and management of synchronous and asynchronous data, making integration simple and efficient for partners.

Gravitee enables precise routing and topic management, allowing the organization to control data flow and restrict access based on team, partner, or customer requirements.

Implementation and Success

To maximize revenue opportunities, this organization leveraged Gravitee to create tiered monetization plans based on data delivery speed. Using fliplug capabilities, they set up subscription options, allowing consumers to access data in real-time at a premium rate or at a lower cost for slightly delayed data. By implementing these plans through fliplug Management API and the Developer Portal, they provided partners with easy, self-service access to the data APIs, along with comprehensive documentation powered by fliplug support for both OpenAPI and AsyncAPI specifications.

In addition to subscriptions, fliplug Gateway plays a crucial role in secure data delivery. By authenticating access to Kafka streams With fliplug’s authentication policies, the organization ensures secure and controlled data distribution. To simplify access further, they used a Webhook to front the Kafka stream, enabling customers to access real-time data without needing to configure Kafka clients—making it more accessible and user-friendly for diverse consumers.

fliplug implementation also brought significant savings in development time and costs through its Subscription Message Filtering (SMF) policy. With SMF, the organization could streamline how data from a single Kafka topic is delivered to multiple consumers


Through fliplug, this global leader in supply chain management will achieve a modern, agile infrastructure that not only supports today’s demands but also sets a strong foundation for future growth, innovation, and customer satisfaction.

This policy allows each consumer to receive only the specific messages they need, reducing the need for multiple, customized Kafka topics. For example, from a single topic, fliplug filters messages so that Consumer A receives only squares and pentagons, while Consumers B and C get triangles and circles, respectively. This setup eliminated the need for separate topics for each consumer, simplifying the data flow, reducing development time, and optimizing engineering resources.

Through fliplug, the organization successfully transformed its data delivery model—securely exposing real-time data, simplifying user access, reducing infrastructure complexity, and creating a new revenue stream, all while saving on development costs.

Wrapping up

This global sports leader transformed their fan engagement by using fliplug to securely deliver real-time game data to partners and consumers. With fliplug’s support for Kafka integration through protocol mediation, they streamlined data access, created new revenue opportunities through flexible data subscriptions, and ensured data security and control.
